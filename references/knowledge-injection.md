# 지식 유입 프로토콜

**원칙: 해당 Action·Task에 필요한 사례만. 개념적 지식 ✗**

---

## 유입 항목

| 유입 항목 | 필터 | 방식 |
|-----------|------|------|
| 동일 Task 패턴 | 해당 Action 내 동일 Task 유형의 검증된 UI 패턴만 | WebSearch |
| 동일 도메인 상위 앱 | 같은 도메인 상위 앱의 해당 Action 화면만 | WebSearch |
| 플랫폼 가이드라인 | HIG·MD3에서 해당 컴포넌트 섹션만 발췌 | WebSearch / references/ |
| SHE 적용 사례 | 동일 Task에서 SHE가 작동한 실제 앱 사례 | WebSearch / references/ |

---

## 깊이 분기

| 모드 | 조건 | 범위 |
|------|------|------|
| 라이트 | 단순 화면·표준 패턴 | references/ 번들만 |
| 풀 | 복잡 화면·비표준 패턴 | references/ + WebSearch |

---

## 필터 규칙

✗ "일반적으로 ~가 좋다" 수준의 원칙
✗ 해당 Action·Task와 무관한 다른 화면 사례
✓ "이 Task에서 이 사례가 왜 작동했는지" 설명 가능한 것만

**범용 원칙 차단 테스트:** 인용하려는 사례·원칙이 "모든 Task에 대입해도 성립"하면 인용 불가. 각 사례에 **"이 Task에서만 성립하는 이유"**를 1문장으로 명시하라. 명시 불가 → 해당 사례 삭제.
