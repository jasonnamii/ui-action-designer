# SHE Protocol

**SHE = Simple → Hide → Embody(Explain).** 복잡한 시스템을 "겉은 단순하게, 내부는 숨기고, 사용 시점에 드러나게" 설계하는 애플식 인터랙션 원칙.

> 단순함 = 기능이 적음 ❌ / 복잡함을 숨긴 상태 ✔
> "기능을 설명하지 말고, 필요할 때 드러나게 만들어라"

| 단계 | 정의 | 역할 |
|------|------|------|
| **S** (Simple) | 표면을 극도로 단순화 | 처음 보는 UI = 최소 정보만 |
| **H** (Hide) | 복잡한 기능·구조를 숨김 | 학습 비용 제거, 인지 부하 차단 |
| **E** (Embody) | 필요 시 자연스럽게 드러남 | 행동 기반으로 기능을 이해하게 만듦 |

**UX의 본질 = 타이밍 제어:** 언제 보여줄지 · 누구에게 보여줄지 · 어떤 행동 뒤에 보여줄지

## S: 4단계 노출 등급

| 등급 | 노출률 | 기준 | 적용 |
|------|--------|------|------|
| **L1 (25%)** | Action 완료 불가결 Task **1개만** + 최소 내비 | "이것 없으면 Action 자체 불가" | 단일 Action (결제, 촬영). 불가결 Task 2+→L2↑ |
| **L2 (50%)** | 불가결 Task 1개 + 보조 Task 1~2개 | "첫 사용에서 필요" | 탐색+실행 혼합, 멀티스텝 플로우 (피드, 검색, 예약) |
| **L3 (75%)** | 대부분 노출, 고급만 숨김 | "반복 사용자 즉시 사용" | 대시보드, 설정, 편집 |
| **L4 (100%)** | 전체 (SHE 미적용) | "숨기면 효율 저하" | 데이터 테이블, 전문가 도구 |

### 등급 판정 4축

| 축 | ←L1 | L4→ |
|----|------|------|
| Task 빈도 | 1회성·저빈도 | 반복·고빈도 |
| 숙련도 | 초보·일반 | 전문가·파워유저 |
| Task 복잡도 | 단일 Action | 다중 Action 병렬 |
| 오류 비용 | 낮음 (되돌리기 가능) | 높음 (되돌리기 불가) |

**판정:** L4방향 2+→1단계↑ / L1방향 3+→L1확정 / 애매→높은 쪽(더 노출)이 안전

**인접 화면:** 등급 차이 2단계 이내. 3+→중간 전환 화면 삽입 또는 재조정.

### 제안 유형별 등급 조정

| 유형 | 조정 | 이유 |
|------|------|------|
| 표준 | 그대로 | 보수적 |
| iOS 26 | **1단계↓** (더 숨김) | 애플 철학 |
| AOS | 그대로 | MD3 정보 밀도 허용 |
| 창의적 | **L1 고정** | 극단 심플→드러남 설계가 핵심 |

## H: 숨김 트리거

| 트리거 | 동작 | 적합 상황 |
|--------|------|----------|
| 스와이프 | 좌우/상하→숨긴 영역 노출 | 모드 전환, 추가 옵션 |
| 롱프레스 | 컨텍스트 메뉴 | 부가 기능, 편집 옵션 |
| 상태 변화 | 조건 충족 시 자동 노출 | 에러, 완료 |
| 컨텍스트 진입 | 화면/섹션 진입 시 | 서브 메뉴, 상세 설정 |
| 스크롤 | 방향/위치에 따라 | 내비 바, FAB |
| 탭 전환 | 세그먼트 전환 | 카테고리별 콘텐츠 |

**매핑 테이블:** 요소 | 현재 상태 | 숨김 여부 | 트리거 유형 | 트리거 조건

## E: 이중 게이트

**게이트 1 (객관):** 동일 인터랙션 패턴 성공사례 검색. 1+→통과 / 0→"사례 미발견" 보고.
**게이트 2 (형 확인):** 객관 결과 + 드러남 설계안 → 형 승인/수정.
**핵심:** "설명 없이, 행동만으로 이해?" — tooltip/onboarding text 필요→재설계 / 행동 자체로 이해→통과.
