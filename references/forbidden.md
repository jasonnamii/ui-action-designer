## §2. 용어·정의 금지

| 금지 | 대응 |
|------|------|
| Action·Task 혼용 | Action=기능(KPI), Task=단계(퍼널). 1:N |
| "단순함 = 기능이 적음" 해석 | "복잡함을 숨긴 상태" = SHE 본질 |
| "UX = 디자인" 동일시 | UX_GATE = 상호작용 품질, SHE = 타이밍·노출 |

## §3. 입력·재현도 금지

| 금지 | 리스크 |
|------|------|
| 스크린샷 60~75% 재현도로 토큰 확정 | 색상값 렌더링 의존 → 게이트 재검증 필수 |
| Figma MCP 미연결 상태에서 "정확한 토큰" 단언 | 폴백 경고 1회 누락 = 품질 왜곡 |
| PDF 1페이지 ≠ 1화면 매핑 무시 | 화면 단위 파싱 실패 |

## §4. SHE 설계 금지

| 금지 | 대응 |
|------|------|
| L1 Surface 과잉 (다 노출) | 발견성 급락 → 4축 판정 엄수 |
| 기능 "삭제" | SHE는 숨김·접근경로 보존. 삭제 ✗ |
| 숨김 트리거 미명시 | "어떤 행동·상태 뒤 등장?" 필수 답변 |
| UX 원리가 SHE 숨김 결정 덮어쓰기 | SHE 결정 우선 + 조정안만 |

## §5. 제안·산출 금지 (§⑦·§⑧)

| 금지 | 대응 |
|------|------|
| 토큰 범위 이탈 제안 | 재추출 or 재제안 |
| 4방향 중 2+ 동일 구조 | 정보 계층 or 인터랙션 흐름 차별화 강제 |
| PRD 정합성 검증 없이 산출 (PRD 존재 시) | `prd-frames.md` §4 매트릭스 필수 |

| HTML·PRD 파일 산출 후 스크린샷 비교 미실시 | 형 명시 코드 산출 시 필수 |
